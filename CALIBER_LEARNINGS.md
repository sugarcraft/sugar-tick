# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha]** `tools/check-path-repos.php` reports closure cleanliness based on actual filesystem path-repos (`{type:path, url:"../<dep>"}` entries), not just `require` keys — so adding `"sugarcraft/candy-vcr": "@dev"` without a matching path-repo entry will pass `composer install` (via the parent `repositories[]` rollup at root) but fail check-path-repos. Both must be added.
- **[gotcha]** `awalsh128/cache-apt-pkgs-action@v1` cannot reliably restore packages whose post-install scripts register `update-alternatives` symlinks (e.g. ffmpeg's transitive `libblas`). Even with `execute_install_scripts: true` the action's `dpkg --configure` does NOT recreate update-alternatives entries. Fix: skip the cache action — direct `sudo apt-get install -y --no-install-recommends <pkg>` is ~15s and always runs real post-install scripts.
- **[pattern]** Session handoff doc convention: when the user says "update the local md file with our current status and options and progress so i can start a new session", write to `plans/<topic>-status.md` AND a paired `plans/session_handoff.md` (actionable starter vs. encyclopedia). `session_handoff.md` opens with a literal copy-paste bootstrap prompt for the new session. Include explicit run IDs so the next session can spot-check freshness.
- **[gotcha]** `gh run list --workflow=ci.yml --branch=master --limit=1` returns the MOST RECENT run regardless of state — a freshly-pushed merge commit shows up as `queued` with empty `conclusion` field. To confirm CI is actually green, either bump `--limit=5` or filter via `gh run list --workflow=ci.yml --branch=master --status=success --limit=1`.
- **[gotcha]** ALL `gh` subcommands fail with `Bad credentials (HTTP 401)` when `GITHUB_TOKEN` is set in the environment to an invalid/expired token — `gh` prefers the env var over its stored keyring auth. Fix: `unset GITHUB_TOKEN && gh ...` once at the start of any session that uses `gh`.
- **[gotcha]** When writing a test stub `class Fake<X> implements <Interface>`, the method signatures must match the INTERFACE byte-for-byte including parameter names, default values, and nullable markers — a mismatch passes syntax but fails at class-load with a `Declaration of ... must be compatible with ...` warning that PHPUnit's `failOnWarning=true` upgrades to a fatal. Always copy-paste the interface signature when writing a stub.
- **[pattern]** PTY-real PHPUnit tests skip cleanly on FFI-less / sandboxed CI via a 5-line `requirePtySyscalls()` preamble: `PHP_OS_FAMILY === 'Windows'`, `extension_loaded('ffi')`, `is_readable('/dev/ptmx') && is_writable('/dev/ptmx')`, `extension_loaded('pcntl')` (only when spawning with `controllingTerminal: true`), and `is_executable('/bin/echo')` (or whichever child binary you spawn).
- **[pattern]** `vendor/bin/phpunit --display-deprecations` reveals the exact source (file:line) of `D`-marked deprecation warnings in test output. Without the flag the test runner just shows the count, not the source.
- **[gotcha]** PTY pump teardown race: when a fast child writes a small payload and exits before the pump's first `tick()`, naively marking the session done as soon as `$child->exited()` returns true causes those bytes to be lost. Fix: watermark the moment the child is first observed exited (`$session->childExitedAt = microtime(true)`), then keep the session live until either master returns 0 bytes (EOF) OR `now - childExitedAt > flushDeadlineSec`. Canonical pattern: `candy-pty/src/Posix/MultiPump.php::tick()`.
- **[gotcha]** PTY pool / FFI-handle pool gating: measure the per-call cost FIRST before committing to a pool architecture. `Libc::lib()` caches the cdef as a per-process static (handle reuse ≈ 0.002ms/call, cold-load ≈ 0.008ms/call, full `PosixPtySystem::open` ≈ 0.05ms/call). The pool's value is BOOKKEEPING (typed `PoolExhaustedException`, idempotent `release()`, `drain()` on shutdown) — NOT handle reuse. Profile before scoping.
- **[gotcha]** Real-PTY tests MUST call `\stream_set_blocking($pair->master()->stream(), false)` before any read on the master — without it, `fread` on the master fd ignores the `$timeout` argument and blocks the whole PHP process until the kernel buffer has data. Symptom of forgetting: a test that should fail with `ExpectTimeoutException` instead hangs forever.
- **[pattern]** When scripting a bash child in a PHP test, wrap the bash body in a quoted heredoc `<<<'SH' ... SH` (note the SINGLE-QUOTED delimiter). The single quotes block PHP variable interpolation so `$user`, `$$`, `$1` land literally in the bash child. Combine with `IFS= read -r <var>` (not bare `read`) when the prompt protocol expects leading whitespace to be preserved.
- **[gotcha]** PHP FFI's `\FFI::cdef()` eagerly resolves ALL declared symbols at library load time — declaring a function the target libc doesn't export (e.g., POSIX 2024 `tcsetwinsize` on glibc < 2.36) causes the entire FFI handle to fail. Gating the declaration behind `if (PHP_OS_FAMILY === 'Darwin')` does NOT help. Fix: build the cdef block conditionally so the declaration is omitted entirely. Canonical: `candy-pty/src/Libc.php::cdef()`.
- **[gotcha]** macOS arm64 + PHP FFI + variadic libc functions = ABI mismatch. Real libc `ioctl(int, unsigned long, ...)` is variadic, and a fixed-arg PHP FFI cdef pushes the third arg into `x2` while the kernel reads it from the stack → ioctl returns `-1` with no useful errno. Workaround on Darwin: route through non-variadic POSIX 2024 wrappers (`tcsetwinsize`/`tcgetwinsize` for TIOCSWINSZ/TIOCGWINSZ) or shell out to `stty -f /dev/fd/<fd>`.
- **[gotcha]** Path-repo `composer.lock` mismatch on PR branches is EXPECTED, not a bug. `candy-core/composer.lock` pins `sugarcraft/candy-pty` to `"version": "dev-master"`. When CI checks out a PR branch where the dep's HEAD is at `ai/<branch>`, composer reads the symlinked path-repo's branch as `dev-ai/<branch>` and rejects the lock — yielding a 7-cell failure pattern on the PR. Do NOT regenerate the lock on the PR branch — that just shifts the failure to the post-merge master push.
- **[gotcha]** `candy-vcr` `Recorder::open($path)` WITHOUT an explicit header writes the JSONL header with the default 80×24 cols/rows — calling `$recorder->recordResize($cols, $rows)` AFTER `open()` lands a resize event but does NOT update the already-written header line. Fix: pass `Recorder::defaultHeader($cols, $rows, $runtimeId)` (or a fully-built `CassetteHeader`) as the second arg to `Recorder::open()` whenever the recorded program runs at non-default dimensions.
- **[gotcha]** `candy-vcr/src/Recorder.php::__construct` writes the JSONL header line eagerly using ONLY the 5 base fields (`v`, `created`, `cols`, `rows`, `runtime`). Adding a new `CassetteHeader` property (e.g. `env`, `timestampMode`) and threading it through `CassetteHeader::__construct` + `JsonlFormat::encodeHeader` is NOT enough — `Recorder::__construct`'s `writeLine([...])` literal must also conditionally append the new field, otherwise the constructed cassette header is missing the value despite the in-memory header object having it correctly set. Symptom: `JsonlFormat::read($path)->header->newField` round-trips as the default even though the `CassetteHeader` object passed to `Recorder::open` had the data. Always cross-check ALL three header writers when extending the schema: `CassetteHeader::__construct`, `JsonlFormat::encodeHeader`, and `Recorder::__construct`.
- **[pattern]** When a PHPUnit test for `--env` / env-capture functionality fails with the captured env appearing empty, the likely culprit is NOT the filter helper but a missed header-write site. Verify by calling the public helper directly via `php -r` to confirm it returns the expected map, then check every place the cassette header is serialized. The bug is downstream of the data layer in 90%+ of cases.
- **[gotcha]** Bash watchdog patterns that `cd` into a lib subdir AND launch `( sleep N && pkill ... ) &` BEFORE the cd command lose CWD context — the watchdog subshell forks early and the foreground `vendor/bin/phpunit` invocation runs in the parent's CWD, not the lib's. Either anchor with absolute paths (`/home/sites/sugarcraft/candy-vcr/vendor/bin/phpunit`) or `cd` first and then launch the watchdog (`cd /path/to/lib && ( sleep N && pkill ... ) & WATCHDOG_PID=$! ; ./vendor/bin/phpunit ...`). Forgetting this yields `vendor/bin/phpunit: No such file or directory` exit 127.
- **[gotcha]** Even with `cd /path/to/lib && ( sleep N && pkill ... ) &` syntax, the multi-line bash watchdog can still fail with `./vendor/bin/phpunit: No such file or directory` exit 127 — observed when the `&` backgrounded watchdog forks while CWD evaluation hasn't completed for the foreground command. SAFEST fix: use the lib's absolute path for phpunit (`/home/sites/sugarcraft/<lib>/vendor/bin/phpunit`) AND anchor every other phpunit-dependent path (cassette tmp file, --bootstrap, --configuration) with absolute paths. Don't rely on CWD persistence across the watchdog fork boundary.

- **[gotcha]** Tracking "has this float been initialized?" with `$prevField > 0.0` is fragile when the field is computed from `round(microtime(true) - $startTime, 3)` — the first event in the first millisecond of a recording legitimately produces 0.0, and the guard then mistakes it for "never set". Use a separate `bool $hasPriorEvent = false` flag instead. Symptom from this anti-pattern: the second event's gap-detection silently skips its first invocation, breaking any `tRaw`/idle-trim arithmetic that depends on it. Canonical fix: `candy-vcr/src/Recorder.php::writeEvent()`.
- **[gotcha]** Real-PTY tests that spawn a child with a `sleep N` between visible outputs MUST give `RecordCommand` a non-EOFing stdin — passing `\fopen('/dev/null', 'r')` EOFs immediately, and `PumpOptions::stdinEofGraceSec` (default 300ms) closes the master before the child wakes from sleep. Symptom: cassette contains only the pre-sleep bytes; the post-sleep `printf 'post\n'` is never recorded. Fix: `[$r, $w] = \stream_socket_pair(STREAM_PF_UNIX, STREAM_SOCK_STREAM, STREAM_IPPROTO_IP); $cmd = new RecordCommand($r);` and keep both ends open for the duration of the test. Close both in the `finally`.
