# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha]** Streaming-writer classes whose `close()` method calls `fclose($this->fh)` (e.g. `candy-vcr`'s `Recorder::close()`) make the underlying stream resource invalid — calling `rewind($fh)` afterwards in the same test throws `TypeError: rewind(): supplied resource is not a valid stream resource`. Test pattern: do NOT call `$r->close()` if you also want to `rewind`/`fgets`/`stream_get_contents` the same `php://memory` handle to inspect the bytes. Either skip the close in tests that need to read back, or close once and then re-open via the file path.
- **[pattern]** When PHPUnit failure messages alone don't explain a Program/Recorder/lifecycle test failure, stage a standalone debug script at `/tmp/dbg.php` with `echo "tick $count: " . get_class($msg)` inside `update()`, run with absolute-path autoload, and inspect the printed Msg sequence. Faster than reading the 800-line Program.php dispatch. Remember to use `require_once '/home/sites/sugarcraft/<lib>/vendor/autoload.php'` (absolute), not `__DIR__ . '/vendor/autoload.php'`.
- **[pattern]** Tee/sink classes attached to `Program` (recorder, debug tap, mirror) should make every `record*`/`write*` method **no-op after close()**, NOT throw `LogicException`. Reason: the runtime has `dispatch(QuitMsg) → close → teardownTerminal → writeOutput(...) → sink->record...` ordering — teardown's environmental writes can't be intercepted by the caller, so a throw-on-closed sink would crash on every clean shutdown. Inverse: `Program::dispatch(QuitMsg)` must call `recorder?->recordQuit(); recorder?->close();` immediately (not at end of `run()`), so the cassette ends on a clean `quit` line and the no-op-after-close drops the teardown noise.
- **[gotcha]** PSR-4 autoload requires one class per file with basename = class name. Putting test helper/fixture classes (e.g. `UserJsonableMsg`, `BadJsonableMsg`) at the bottom of a `FooTest.php` works ONLY because PHPUnit `require`s `FooTest.php` to instantiate the test, side-effect-loading the inline classes. If a SECOND test file (`BarTest.php`) references those fixtures and runs first in isolation (e.g. `phpunit --filter BarTest`), autoload fails — there is no `tests/.../UserJsonableMsg.php` for the autoloader to find. Move shared fixtures into their own PSR-4-conformant files (`tests/Msg/UserJsonableMsg.php`) from the start when more than one test will use them.
- **[pattern]** For a `Registry`-of-pluggable-strategies design (e.g. candy-vcr's `Msg\Registry` of `MsgSerializer`s), use unqualified class names as the routing tag for builtins (`KeyMsg`, `WindowSizeMsg` — readable cassettes/dumps) and FQCN for the catch-all that handles user types (avoids cross-namespace collision). The two coexist if the builtin's `canDecode()` checks against an explicit allowlist (`classForTag()` match expression) so it never accidentally claims a user FQCN. Catch-all serializer (`JsonableSerializer`) round-trips via `new $class(...$data)` named-arg unpacking — works when constructor parameter names match the keys returned by `jsonSerialize()`.
- **[gotcha]** `gh pr view <n> --json merged` fails with `Unknown JSON field: "merged"` — that field doesn't exist. Use `--json state,mergedAt` instead: a merged PR returns `{"state":"MERGED","mergedAt":"<timestamp>"}`, an open PR returns `"mergedAt":null`. The error output helpfully lists the available fields but the immediate fix is `mergedAt` not `merged`.
- **[gotcha]** `candy-core`'s `setupTerminal()` emits output bytes (e.g. `\x1b[?2027h` unicodeOn, alt-screen-enter, cursor-hide) BEFORE the startup `WindowSizeMsg` dispatch, and `teardownTerminal()` emits closing counterparts (e.g. `\x1b[?2027l`) AFTER `dispatch(QuitMsg)` once `loop->stop()` has fired. A `Recorder` (or any `writeOutput` tee) MUST stay open across teardown — closing it on `QuitMsg` drops those teardown bytes from the cassette and then a fresh `Program` on replay emits them, producing a byte-mismatch at the tail. Close the recorder at the END of `Program::run()`, not inside the `QuitMsg` branch.
- **[pattern]** Tee/sink classes attached to `Program` (recorder, debug tap, mirror) should make every `record*`/`write*` method **no-op after close()**, NOT throw `LogicException`. Reason: the runtime has `dispatch(QuitMsg) → loop->stop → teardownTerminal → writeOutput(...) → sink->record...` ordering — teardown's environmental writes can't be intercepted by the caller, so a throw-on-closed sink would crash on every clean shutdown.
- **[gotcha]** A `candy-vcr` `Player` driving a `Program` in `SPEED_INSTANT` mode must NOT schedule every cassette event at `delay=0` via `addTimer` — React's `StreamSelectLoop` fires them all in a single tick before the program's framerate-based render tick gets a turn, so `view()` is never rendered and no output bytes are produced (`outputCount: 0` despite `outputCount > 0` in the cassette). Schedule events sequentially: dispatch one, then `addTimer($yield, $next)` with `$yield ≈ 5 ms` between events (`Player::INSTANT_YIELD_SECONDS`); in `SPEED_REALTIME` use `events[i+1]->t - events[i]->t`, clamped to `>= 0`. Same trap: replay programs need a high `framerate` (e.g. 1000.0 in `ProgramOptions`) so the renderer wakes up on the yield interval rather than the default ~60 Hz.
- **[gotcha]** A `candy-vcr` round-trip test (record session A → replay into fresh Program B → assert byte-equal) is NOT achievable with `ByteAssertion` alone — recording captures whatever the framerate-based renderer emitted during the recording loop's lifespan, the replay loop runs over a different chained-timer timeline (Player yields between events to let render fire) and emits a *cell-grid equivalent* but byte-different stream. Tests should assert `programQuitCleanly` + event-count tallies for PR4 and defer true byte-of-bytes round-trip parity to a `ScreenAssertion` (cell-grid via candy-vt) in PR5. Symptom: `byte mismatch: expected 41 bytes, got 115 bytes; first divergence at offset 32` between the recorded `tick: 6` frame and a replay that emitted `tick: 3` plus extra `\x1b[?2026h.[?2026...` synchronized-update bytes.
- **[gotcha]** `program->send($msg)` on `candy-core` `Program` enqueues the Msg directly into the dispatch queue, BYPASSING the input parser — the recorder's `recordInputBytes` hook never sees those Msgs and they don't land in the cassette as `input` events. To exercise the recorder's input path in tests, write actual bytes through the input stream resource: `$loop->futureTick(fn() => fwrite($writer, 'abc'));` where `$writer` is the write end of the `stream_socket_pair` whose read end was passed to `ProgramOptions(input: $reader, ...)`. Test models that need to keep their loop alive long enough for the recorder to capture frames should set `quitAfter: PHP_INT_MAX` and drive quit externally with `$loop->addTimer(0.020, fn() => $program->quit())`.
- **[pattern]** When a `ByteAssertion`-style diff helper produces a hex window starting at the first-divergence offset (not at offset 0), tests asserting on the diff string cannot expect the FULL hex of either expected/actual — the window only contains the divergent tail. For `("\x1b[2J", "\x1b[2K")` whose first three bytes `\x1b[2` are equal, the window starts at offset 3 and shows only `4a (J)` / `4b (K)`. Force the divergence to offset 0 (e.g. `str_repeat('A', 16)` vs `str_repeat('B', 16)`) when you want to assert the full 16-byte hex window (`4141…41` / `4242…42`).
- **[gotcha]** The Bash tool's working directory persists across separate Bash calls. After `cd /home/sites/sugarcraft/<lib> && cmd`, the NEXT Bash call still starts in `<lib>` — `git status` will then show paths with `../` prefixes that look like they're outside the repo but aren't. A bare `ls candy-core/src/` from inside `candy-core` returns NO output (empty `{}` response from the tool, not a `No such file or directory` error) because the path is relative to the wrong cwd. Bare `grep candy-core/src/Cmd.php` after `cd .../candy-vcr` produces `ugrep: warning: candy-core/src/Cmd.php: No such file or directory` (and same for `grep candy-vcr/src/Player.php` after `cd /home/sites/sugarcraft/candy-vcr` — the file is at `src/Player.php` from inside that cwd). Always use absolute paths or prefix every Bash call with `cd /home/sites/sugarcraft && ...` to anchor it. Same trap bites HEREDOC-written `/tmp/dbg.php` scripts: `require_once __DIR__ . '/vendor/autoload.php'` resolves `__DIR__` to `/tmp/`, NOT to the working dir of the heredoc-emitting Bash call. Use absolute autoload paths (`require_once '/home/sites/sugarcraft/<lib>/vendor/autoload.php';`) in any standalone PHP script you stage under `/tmp/`.
- **[gotcha]** When a new lib subdir is added, the caliber pre-commit hook does NOT auto-stage untracked files in the new dir. Stage every new file explicitly: `git add candy-<slug>/composer.json candy-<slug>/phpunit.xml candy-<slug>/README.md candy-<slug>/CALIBER_LEARNINGS.md candy-<slug>/src/...` plus the cross-cut edits. `git status -uall <subdir>/` enumerates the per-file list when you forget what you created.

- **[pattern]** Cross-lib hook seams that need to live in candy-core but are implemented in a downstream lib (e.g. recorder lives in candy-vcr but the tee point is `Program::writeOutput()` in candy-core) take the form: candy-core ships an INTERFACE, `Program::withRecorder(?Recorder $r): self` accepts the interface, and the downstream lib provides the concrete class implementing it. Avoids candy-core depending on candy-vcr (the dependency goes the other way) and lets other consumers (debug tap, network mirror, crash bundle) slot into the same hook without pulling cassette plumbing.

- **[gotcha]** When a Player replays a recorded cassette into a FRESH `Program`, both the cassette AND the replay program emit a startup `WindowSizeMsg` (cassette's first `resize` event was the original tty.size() lookup; the new program does its own tty.size() lookup at `setupTerminal()` time). Dispatching the cassette's first resize on replay duplicates the msg, throws the model's tick count off by one, and produces a single-cell screen diff that looks like flake but is deterministic (e.g. `(0,6) expected '6' actual '7'` for a counter-incrementing model). Skip the first `resize` event on replay (`$firstResizeSkipped` flag) — subsequent resizes from in-session SIGWINCH events still need to be dispatched.
- **[gotcha]** When threading a new `&$flag` reference variable through nested closures in PHP, you must add it to EVERY `use()` clause along the path — adding `&$firstResizeSkipped` to the inner `dispatchEvent` closure but forgetting the outer `$step = function () use (...)` closure produces `TypeError: Argument #N (...) must be of type bool, null given` at runtime, NOT a compile-time error. The variable resolves to `null` in the outer scope and gets passed down. Audit every nested `use(...)` when adding a closure-captured variable; `grep -n "use (" Player.php` lists them all in one pass.
