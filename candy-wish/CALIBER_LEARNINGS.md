# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha]** Avoid `pcntl_fork()` inside PHPUnit tests — the PHP runtime carries opcache/mock state/FFI handles into the fork and tests hang silently (PHPUnit produced zero output for minutes). Spawn a small fixture PHP script (e.g. `tests/Foo/_fixtures/runner.php`) via `proc_open()` instead and drive its stdin/stdout pipes from the test.
- **[pattern]** When testing PTY-spawning code (`InProcessTransport`, `candy-pty` consumers), guard tests with `requirePtySyscalls()` that skips on missing ext-ffi / ext-pcntl / unreadable `/dev/ptmx` — these are non-negotiable host requirements and CI must skip cleanly rather than fail.
- **[gotcha]** `Pty::read()` distinguishes `null` (timeout / no data yet) from `''` (EOF) — treat them differently. The pump loop's master-side reader should keep polling on `null`/`''` and only exit when the child has exited; do not bail on a single empty read or you'll race child output.
- **[gotcha:linux-no-auto-sighup-on-master-close]** Closing the PTY master in PHP/Linux does NOT auto-deliver `SIGHUP` to the slave-side child via lost-ctty (BSD-style behaviour I assumed). Children that ignore STDIN EOF (`sleep`, daemons, anything not reading stdin) keep running until reaped. The teardown sequence that actually works on Linux: (1) on stdin EOF / EPIPE / pump exit, send `SIGHUP` explicitly via `posix_kill($child->pid, SIGHUP)`, (2) wait ~200 ms for graceful exit, (3) `SIGKILL` if still alive, (4) close the PTY master, (5) `child->wait()` reaps. Without the explicit `posix_kill`, `wait()` blocks for the child's natural duration (5+ seconds for `sleep 5`) instead of the expected sub-second teardown. Canonical implementation in `InProcessTransport::runChild()`'s finally block.
- **[env]** Always wrap potentially-blocking `vendor/bin/phpunit` invocations with `timeout 60` (or longer) when developing PTY/IPC code — a hung pump loop produces zero output, no error, and you can't tell a hang from a slow test without the timeout.
- **[convention]** New i18n strings for a candy-wish error message must land in `lang/en.php` keyed under a dotted namespace (`transport.bad_stdin`, `bubbletea.bad_factory`, etc.) and looked up via `Lang::t('namespace.key')` — do not throw raw English strings from PHP.
- **[convention]** When `Session->cols`/`->rows` are 0 (ForceCommand deployments without COLUMNS/LINES set), fall back to 80×24 before passing to `Pty::spawn()` — caveat 5 in `plans/plan-candy-wish-pty.md`. A 0-width PTY confuses curses-based clients.
- **[gotcha]** Tests that signal a `proc_open()`-spawned PHP supervisor (e.g. driving SIGWINCH into `InProcessTransport`) are racy if you `usleep(150_000)` to wait for "child has started". PHP cold-start + autoloader + `Pty::spawn` can run 200–400 ms in CI, so the supervisor's signal handler is sometimes installed AFTER the test's `posix_kill(...,SIGWINCH)` — losing the first SIGWINCH entirely and only the post-mutation state is observed. Fix pattern: poll the child's side-effect file (e.g. an `stty size` log) until the FIRST baseline sample lands, then mutate state + signal, then poll for the post-state sample. Never use a fixed sleep as the synchronisation primitive in signal-driven tests.
- **[gotcha]** Manual signal-delivery debugging via `timeout 3 php fixture.php ... & SUP_PID=$!; kill -WINCH $SUP_PID` does NOT reach the supervisor — `kill` targets the `timeout` wrapper PID, not the inner PHP process, and `timeout` does not forward arbitrary signals. Drop the `timeout` wrapper (or pull the inner PID from `pgrep -P $SUP_PID`) when reproducing signal-handling bugs at the shell.
- **[gotcha]** `gh pr merge <n> --merge --delete-branch` immediately after `git push` of a follow-up commit can fail with `GraphQL: Pull Request is not mergeable` — GitHub's mergeability check hasn't re-evaluated yet. Wait a few seconds and retry the same `gh pr merge` call; do not switch to `--admin`, force-push, or rebase as a workaround.
