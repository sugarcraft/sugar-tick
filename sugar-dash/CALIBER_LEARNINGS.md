# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha:project]** sugar-dash `Console` keeps DUAL state: `private array $entries` (filtered, view cache) AND `private array $originalEntries` (source of truth). Most withers reconstruct via `new self(entries: $this->originalEntries, ...)` which means any wither that ADDS/REMOVES entries MUST mutate `originalEntries`, not the filtered `entries` array. Symptom: `Console::new()->withEntry($e)->withShowPrefix(false)->render()` returns empty string because `withShowPrefix` rebuilt from `originalEntries` (still empty). Pattern: in `withEntry()` do `$clone->originalEntries = array_merge($clone->originalEntries, [$entry]); $clone->entries = $this->filterEntries($clone->originalEntries);`. Apply this audit to any class with both filtered and unfiltered collections.
- **[gotcha:project]** `assertSame('#F9FAFB', strtolower($actual))` ALWAYS fails because the expected literal stays uppercase while the actual is lowercased — the comparison is asymmetric. Either pre-lowercase the literal (`assertSame(strtolower('#F9FAFB'), strtolower($actual))`) or compare uppercase both sides. Grep `tests/Grid/` for `assertSame\('#[A-F]` to find latent failures of this form.
- **[gotcha:project]** When a wither uses `return new self(color: $color, fillColor: $this->fillColor, ...)` to dodge the readonly problem, it ONLY copies state that's a constructor-promoted property. Non-promoted instance state — typically `private array $dataPoints = []`, `private ?DendrogramNode $root = null`, accumulators set via `withRoot()` / `withDataPoints()` / `withPoint()` — is silently dropped. Symptom: `Dendrogram::new()->withRoot($node)->withNodeColor(...)->render()` produces empty output. Fix: drop `readonly` on the ctor-promoted Color/style params and switch every wither back to clone-then-mutate. When auditing one wither in a Grid file, audit ALL withers in that file — the same pattern is replicated across all color/style mutators. Known files: `Area.php`, `Bubble.php`, `Dendrogram.php`, `Funnel.php`, `OHLC.php`, `Partition.php`, `PERT.php`, `Sankey.php`, `Sequence.php`, `State.php`, `Sunburst.php`, `Treemap.php`, `Waterfall.php`, `CandlestickChart.php`. Combined corollary: do NOT mark constructor-promoted color/style properties `readonly` on classes that have `withX()` clone-mutate withers — `Cannot modify readonly property` fires at runtime even with `clone`.
- **[gotcha:project]** Value-object helper classes (`DendrogramNode`, `PertTask`, `SunburstSegment`, etc.) hit the same readonly trap independently. Their `withColor()` / `withDuration()` etc. must NOT assign to readonly promoted props on a clone — either drop `readonly` on those props or rebuild via `new self(...)` (which for these small DTOs is safe since they have no non-promoted state).
- **[pattern:project]** When extracting an inline secondary class to its own PSR-4 file (e.g. `OHLCPoint` out of `OHLC.php`, `TreemapLeaf` out of `Treemap.php`), do TWO edits: (1) `Write` the new `src/Grid/<Name>.php` with the full class body plus `<?php`, `declare(strict_types=1);`, `namespace SugarCraft\Dash\Grid;`, and any `use` statements the class needs (esp. `SugarCraft\Core\Util\Color`); (2) `Edit` the original file to delete the inline class declaration including its preceding docblock, leaving only the owning class. Forgetting step (2) leaves a duplicate-class declaration that fatals at autoload time; forgetting `use Color` in step (1) gives `Class "Color" not found` at first instantiation.
- **[gotcha:project]** sugar-dash chart label-row rendering must compute chars-per-column from the chart width: `$perPoint = max(1, intdiv($chartWidth, max(1, $dataCount)))` then `mb_substr($label, 0, $perPoint)` + `str_pad($label, $perPoint)`. Naive `mb_substr($label, 0, 1)` or hardcoded `mb_substr($label, 0, 2)` collapses every label to one or two characters, so `assertStringContainsString('Jan', $rendered)` or `assertStringContainsString('AAPL', $rendered)` fails with only `J` or `AA` visible. Same trap in `Area::render()`, `Bubble::render()`, `OHLC::render()`. For OHLC-style charts where each point gets a `$barWidth + $spacing` cell, use `$cellWidth = $barWidth + $spacing` as the truncation length, not the narrower `$barWidth`. Also: `Bubble::render()` was passing a float into `str_pad()` — always `(int)` the width param.
- **[gotcha:project]** sugar-dash `render()` methods often have BOTH an outer width/height/empty guard AND an inner `if ($count === 0) return '';` inside the private `renderDiagram()` helper (seen in `Sequence.php`, `State.php`, likely also `Class*`, `PERT`, `Sankey`). Removing only the outer guard leaves the inner one to silently produce `''` for empty collections — tests still get empty string. Either delete the inner return too, or replace it with chrome rendering: `[$tl,$tr,$bl,$br,$h,$v] = $this->getStyleChars(); return $top . str_repeat("\n".$mid, $height-2) . "\n" . $bot;`. Grep for `if (\$\w+Count === 0)` in any Grid file you're auditing.
- **[gotcha:project]** sugar-dash `Editor::render()` short-circuits to `''` when `$contentWidth = $useWidth - $lineNumberWidth - 2 <= 0`. With `showLineNumbers = true` (the default), `calculateLineNumberWidth()` returns `max(2, strlen(count($lines)))` + 1 for separator, so `setSize(5, 3)` produces `contentWidth = 5 - 3 - 2 = 0` and renders empty — breaking `testRenderWithVerySmallSize`. Either auto-disable the line-number gutter when `useWidth` is too small (`$showLineNumbers = $showLineNumbers && $useWidth >= $lineNumberWidth + 4`) or downgrade the empty short-circuit to render just chrome. Same pattern lurks in any Grid component that subtracts a fixed gutter/margin from the requested width.
- **[gotcha:project]** sugar-dash chart components with a fixed default-height fallback (`$this->height ?? 20`) silently clip the trailing items when collection size exceeds the canvas. `Sankey::render()` with `useHeight = 20` and three default-spacing nodes drops the third — `testAddMultipleNodes` fails on `assertStringContainsString('Node 3', $rendered)`. Default height MUST be derived from collection size: `$this->height ?? max(20, count($this->nodes) * ($this->nodeSpacing + 5) + 8)`. Audit every `?? 20` / `?? 10` / `?? 15` literal in `Grid/*.php` `render()` methods and replace with a `max(default, count(...) * row_height)` formula.
- **[gotcha:project]** Treemap-style cell layout that advances `$currentX += $cellWidth + 1;` (with a one-column gap) overflows available width and silently drops subsequent leaves. Cells must butt directly against each other: `$currentX += $cellWidth;` with no `+1`. Also bump `max(2, intval(...))` to `max(3, intval(...))` so labels have room. The same bug pattern lurks in any `Grid/*.php` layout loop that uses both an inter-cell offset AND a wrap-on-overflow check. ListComponent ellipsis truncation has a related off-by-one: `truncateToWidth($line, $w)` returning `$w` cols then appending `…` overflows by one — budget one column for the ellipsis.
- **[gotcha:project]** PHP 8.4's chained-new (`new self(...)->method()`) does NOT parse on PHP 8.3, which is the project's runtime. Wrap with parens: `(new self(...))->method()`. Grep `grep -nE 'new self\([^)]*\)->' src/Grid/*.php` before committing any factory method that uses this shape.
- **[gotcha:project]** `php -l` won't detect a stray space inside a property declaration like `private $ lifelineColor` reliably across versions — `Sequence.php:91` shipped with this typo and parse-failed on autoload but linted in isolation. When extracting/refactoring promoted constructor params, run `php -l` AND `composer dump-autoload` and try instantiating the class before committing.
- **[gotcha:project]** Arithmetic on `microtime(true) * 1000` and `count($arr) % $float` silently emits `Implicit conversion to int loses precision` deprecations under PHPUnit's `failOnWarning=true` / `failOnDeprecation=true`. Cast intermediates with `(int)` or use `intdiv()` / `fmod()` before passing to `%`. Audit anywhere `microtime() * <number>` or `% $someFloat` appears in `src/Grid/`.
- **[gotcha:project]** Negative array indexes (e.g. `selectedIndex = -1` to mean "none") trigger undefined-array-key warnings under `failOnWarning`. Guard with `isset($arr[$index])` or `$index >= 0 && $index < count($arr)` — `if ($index < count($arr))` alone is HALF a check.
- **[gotcha:project]** Several sugar-dash `render()` methods (`State::renderDiagram()`, `Partition::renderRow()`) reference destructured locals like `$v` (vertical border char) without including them in the `[$tl, $tr, $bl, $br, $h, $v] = $this->getStyleChars();` destructure. Symptom: `Undefined variable: $v` warning under `failOnWarning`. When borrowing a helper that destructures style chars, copy the FULL destructure or extract the chars you need explicitly.

- **[gotcha:project]** sugar-dash `Chart::renderLineChart()` builds its `$grid` top-down (row 0 = top of the chart, row $chartHeight-1 = bottom) but `generateGridLines()` returns Y-axis labels bottom-up (`gridLines[0]` = min, `gridLines[$chartHeight-1]` = max). The Y-axis label loop must invert: `$labelIndex = $chartHeight - 1 - $row;` then `$gridLines[$labelIndex]`. Naive `$gridLines[$row]` puts min at the top and max at the bottom — silently wrong, no test failure if tests only check `assertStringContainsString` on a value. Audit any chart whose grid is top-down but whose label generator is bottom-up.
- **[gotcha:project]** sugar-dash `Code::renderBorder()` uses `str_repeat('─', $contentWidth)` but `renderCodeLine()` wraps content with `'│ ' . $alignedLine . ' │'` — two extra inner-padding spaces. The border must be `str_repeat('─', $contentWidth + 2)` so the corners line up with the `│` of the content rows. When fixing a Grid component's border width, render both border and content rows and visually compare widths; off-by-2 errors hide easily.
- **[gotcha:project]** Replace `max(1.0, abs($range))` with `$range = abs($max - $min); if ($range == 0.0) { $range = 1.0; }` in sparkline/chart range computation. `max(1.0, abs($range))` clamps sub-unit ranges (e.g. 0.5) up to 1.0, flattening the chart for small-delta data; only the zero case actually needs the fallback. Audit `grep -n 'max(1.0, abs' src/Grid/*.php`.

- **[gotcha:project]** `str_pad($line, $width)` counts BYTES not visible columns, so ANSI-prefixed lines like `\x1b[38;2;...m█\x1b[0m` get under-padded (visible width = 1, byte width ~20). Pattern: strip ANSI with `preg_replace('/\x1b\[[0-9;]*m/', '', $line)`, measure visible width via `mb_strlen($visible, 'UTF-8')`, then append `str_repeat(' ', max(0, $useWidth - $visibleWidth))` manually. Was the bug in `Scrollbar::render()`'s width-padding step — caller passed `$useWidth > 1` and got a vertical bar that didn't fill its allocated cell. Audit any `str_pad` in `Grid/*.php` that runs AFTER ANSI codes have been inlined into the line.

- **[gotcha:project]** OHLC-style charts that loop over data points and emit `chartHeight` rows PER point produce `chartHeight × pointCount` rows of output instead of the intended `chartHeight`. Symptom: a 10-row chart with 5 points balloons to 50 rows and breaks every height-related assertion downstream. Fix: compose ALL points onto a single shared grid (`array_fill(0, $chartHeight, str_repeat(' ', $innerWidth))`) with a parallel `cellColors` matrix, then emit ONCE per row, splicing per-cell color transitions back via `Ansi::reset()` / `toFg(...)`. Same shape can hide in any Grid chart that plots multiple per-X data series on a shared vertical canvas.
- **[pattern:project]** Per-cell ANSI coloring on a composed grid: build BOTH the char grid AND a parallel `$cellColors` matrix (`array_fill(0, $rows, array_fill(0, $cols, null))`). In the emit loop, track `$currentColor` and only output `Ansi::reset()` + new `toFg(...)` when `$cellColors[$row][$col] !== $currentColor`. This minimizes ANSI noise, keeps the column count consistent for downstream width assertions, and avoids the per-row color reset that bloats output.
- **[pattern:withState]** `BaseModule::withState(array $overrides): static` — clone-mutate helper for immutable module state stored in a private `$state` array. `$clone = clone $this; $clone->state = array_merge($this->state, $overrides); return $clone;`. Avoids the readonly-property-wither trap by keeping state in a non-promoted private array and using `clone` rather than `new self(...)` reconstruction. Subclasses call `withState()` in their `update()` to produce the next module instance.
- **[pattern:grid-dir-deletion]** When an entire subdir like `Grid/` is deleted and classes moved to parent `Layout\`, the old import path `use Layout\Grid\{A, B}` in README examples and doc-site quickstarts goes stale — it now refers to nothing. Example: `use SugarCraft\Dash\Layout\Grid\StackedGrid` needs to become `use SugarCraft\Dash\Layout\StackedGrid`. Always grep `README.md` and `docs/lib/<slug>.html` for `\<Grid\` after any subdir deletion in sugar-dash. (step 03.03 Grid/ 44-file deletion surfaced this.)
- **[pattern:dual-foundation-ssot]** sugar-dash Foundation types are NOT duplicates of canonical — step 03.05 investigation: `Color` (true duplicate, class replaced with `class_alias` shim → Core\Util\Color), `StyleParser` (same parsing logic but produces Dash\Foundation\Cell/Style, NOT Sprinkles\Cell/Style — NOT drop-in; kept as sugar-dash SSOT because `$cell->style->foreground->r` access pattern requires public readonly Color property on Style, which Sprinkles\Style does not expose). The 5 retained types (Style/Theme/Rect/Buffer/Cell) are intentionally distinct from same-named canonical types in candy-sprinkles/candy-core/candy-vt due to different upstream lineage (inline-termui vs lipgloss/ratatui/VT-emulator).
- **[pattern:dual-style-ssot]** sugar-dash Style (toAnsi, public foreground/background Color slots) vs Sprinkles Style (lipgloss padding/margin/borders, private $fg/$bg). Do NOT alias one to the other — the accessor patterns differ: Dash consumers do `$style->foreground->r`, Sprinkles consumers call `$style->foreground(?Color)` as a setter.
- **[pattern:dual-theme-ssot]** sugar-dash Theme (10 colour slots + bar/text/fg/bg/color/highlight helpers) vs Sprinkles Theme (13 colour slots: adds muted/info/border/separator/cursor, readonly properties only, no helpers). Both are canonical for their lib.
- **[pattern:dual-rect-models]** sugar-dash Rect (minX/minY/maxX/maxY — rectmath bounds model) vs Core\Rect (x/y/width/height — ratatui offset+size model). Choose by which upstream semantics the consumer needs.
- **[pattern:dual-buffer-roles]** sugar-dash Buffer (Sizer/Drawable, immutable ANSI renderer for inline-termui layout) vs candy-vt Buffer (mutable VT-output grid for terminal emulation). Different responsibilities — do not conflate.
- **[pattern:dual-cell-shapes]** sugar-dash Cell (rune + Style for inline-termui renderer) vs candy-vt Cell (grapheme + Sgr + continuation + hyperlink for VT emulation). Both canonical for their lib.
- **[pattern:dual-ring-queue]** NotificationQueue uses two-slice semantics (not a true ring buffer) for the dual-ring pattern: `items[max 20]` (active, dismissable) + `history[max 50]` (append-only). At small max sizes the overhead of a true ring buffer is unnecessary; two plain PHP arrays with `array_shift`/`[]=` achieve O(1) amortized push and O(1) dismiss. When items ring hits capacity, oldest is evicted to history before new push. When history hits capacity, oldest is evicted. Do NOT use `list` type hint (PHP 8.4 only — project is PHP 8.3); use `array` with `@var list<T>` doc annotation.
- **[pattern:atomic-tmp-rename]** `State\Persistence::save()` uses the Homestead atomic-save pattern: `file_put_contents($tmp, ..., LOCK_EX)` then `rename($tmp, $path)`. The rename is atomic on POSIX filesystems — the target file is never in a partially-written state even if the process crashes between the write and the rename. On Windows the rename is not guaranteed atomic; the exception handling cleans up the temp file on failure. Always use this pattern for user-facing state files rather than direct `file_put_contents($path, $data)`. (step 03.12 StateMachine split surfaced this.)
