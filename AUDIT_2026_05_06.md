# Source-app coverage audit — 2026-05-06

A line-by-line comparison of every PHP port against its upstream Go counterpart.
Each section lists **API surface gaps**, **examples missing**, and
**documentation gaps** found by automated audit agents. The table is the
backlog for the next port wave.

This is a snapshot, not a contract — items get fixed (or explicitly deferred)
as PRs land. When an item is closed, mark it ✅ inline and link the PR.

> Audited libraries: candy-core, candy-sprinkles, sugar-bits, candy-shell,
> candy-shine, sugar-prompt, sugar-charts, honey-bounce, candy-zone.

> Wider port wave (super-candy, sugar-tick, candy-mines, sugar-stash,

> candy-freeze, candy-mold, candy-kit, candy-metrics, sugar-crush,

- **TextInput:** `VirtualCursor` / `SetVirtualCursor`, `Styles` / `SetStyles`, `Width()` getter, `Position()`, `Focused()`, `AvailableSuggestions()`, `CurrentSuggestionIndex()`, `Blink` cmd, `Cursor()` accessor, public `KeyMap` field.
- ✅ **TextArea navigation + insertRune + word + setPromptFunc.** (Backlog #23)
  Added `cursorUp`/`cursorDown`, `moveToBegin`/`moveToEnd`,
  `pageUp`/`pageDown` (uses `height`; falls back to one row when
  height=0), `insertRune` (multibyte safe), `word()` (current word
  under cursor), `setPromptFunc(fn(int $row, string $line): string)`
  (wins over the static `withPrompt()`). Still TODO:
  `ScrollYOffset` / `ScrollPercent` / `Styles` / `SetStyles` /
  `VirtualCursor` / `Focused()` / `Cursor()` / `Line()` / `Column()` /
  `Width()` / `Height()` getters.
- **Viewport:** ✅ horizontal scroll shipped — `xOffset` / `setXOffset` / `scrollLeft` / `scrollRight` / `withHorizontalStep` / `horizontalScrollPercent` / `atLeftmost` / `atRightmost`, `←`/`h` and `→`/`l` key bindings, `view()` drops left cells via `Util\Width::dropAnsi` so SGR state is preserved. (Backlog #22) Still TODO: `SoftWrap`, `FillHeight`, `LeftGutterFunc`, `StyleLineFunc`, `HighlightStyle` / `SelectedHighlightStyle`, `SetContentLines`, `GetContent`, `EnsureVisible`, `PastBottom`, `SetHighlights` / `ClearHighlights` / `HighlightNext` / `HighlightPrevious`, separate `SetWidth` / `SetHeight`.
- **List / ItemList:** ✅ `setItem` / `insertItem` / `removeItem` / `items()` shipped (audit #21, partial). Still TODO: `SetDelegate` (no delegate concept at all), `MatchesForItem`, `GlobalIndex`, `Cursor()` (page-local), `CursorUp` / `CursorDown`, `GoToStart` / `GoToEnd`, `PrevPage` / `NextPage`, `Select(i)`, `ResetSelected`, `ResetFilter`, `FilterState` / `FilterValue` / `SetFilterText` / `SetFilterState` / `SettingFilter` / `IsFiltered`, `AdditionalShortHelpKeys` / `AdditionalFullHelpKeys`, custom `Filter` func, `SetStatusBarItemName`, `SetSpinner` / `ToggleSpinner` / `StartSpinner` / `StopSpinner`, `DisableQuitKeybindings`, `ShortHelp` / `FullHelp`, `SetShowPagination`.
- **Table:** `Rows()`, `Cursor()` / `SetCursor`, `UpdateViewport`, `HelpView`, `FromValues`, `Focused()`, separate `SetWidth` / `SetHeight` and getters, public `Help` / `KeyMap` fields.
- **Paginator:** `ItemsOnPage`, `ArabicFormat`, `SetTotalPages`, public `KeyMap`.
- **Progress / AnimatedProgress:** `WithColors(...colors)` (multi-color blend beyond start/end), `WithColorFunc` (dynamic), `WithScaled` (scale gradient to fill), `PercentageStyle` field, `SetSpringOptions`, `IsAnimating()`, `SetWidth` / `Width()`, `Init()`.
- **Spinner:** `ID()`. Public `Spinner` / `Style` fields not exposed for direct mutation.
- ✅ **Help** — `showAll(bool)`, `width(int)` / `getWidth()`,
  `view(KeyMap)` unified entry, `withEllipsis(glyph)` truncation char,
  `withFullSeparator(string)` row-joiner, `shortHelpView([]Binding)`
  / `fullHelpView([][]Binding)` raw-binding entry points all shipped.
  (Backlog #20 partial.) Still TODO: `Update`, `Styles` struct.
- **Key/Binding:** `SetKeys`, `Keys()`, `SetHelp`, `Help()`, `Enabled()` / `SetEnabled`, `Unbind()`, top-level `Matches(key, ...bindings)`, `WithDisabled`.
- **Timer:** `ID()`, `Toggle()`, `Timeout` / `Interval` field exposure.
- **Stopwatch:** `ID()`, `Toggle()`, `Elapsed()`, `Interval` field exposure.
- **Cursor:** `Mode()` getter, `Blink` cmd, `Focus()` cmd-return, `BlinkSpeed` / `IsBlinked` exposure, `TextStyle` / `Style` fields.
- **FilePicker:** `DidSelectFile` / `DidSelectDisabledFile` (canonical selection API), `HighlightedPath()` (we have `highlightedEntry` returning Entry only), `ShowPermissions`, `AutoHeight`, `Height()`, public `Path` / `CurrentDirectory` / `Cursor` / `FileSelected` / `Styles` / `KeyMap` exposure, `Init()`.

### Missing examples

`textinputs` (multi-field form), `chat` (textarea chat input), `list-default` / `list-fancy` / `list-simple` (we have only `item-list.php`), `progress-animated` vs `progress-static` split (we have one `progress.php`), `spinners` plural (multi-spinner picker), `pager` (viewport pager demo).

### Doc gaps

- No per-component API reference / option tables (PHP needs explicit method listings since there's no godoc).
- The `Styles` configuration model that pervades bubbles upstream is largely absent in our port.
- No `KeyMap` customisation docs.
- Quickstart only covers TextInput and AnimatedProgress — no snippets for List, Table, Viewport, FilePicker, Help.
- README "14 components" line lists `AnimatedProgress` separately, totalling 15 entries against upstream's 13.
- No documentation of the `Msg` types each component emits (`BlinkMsg`, `TickMsg`, `SpringTickMsg`, `StartStopMsg`, `TimeoutMsg`).
- Filter delegate / item delegate concept undocumented (and seemingly unimplemented) for `ItemList`.
