# Source-app coverage audit — 2026-05-06

A line-by-line comparison of every PHP port against its upstream Go counterpart.
Each section lists **API surface gaps**, **examples missing**, and
**documentation gaps** found by automated audit agents. The table is the
backlog for the next port wave.

This is a snapshot, not a contract — items get fixed (or explicitly deferred)
as PRs land. When an item is closed, mark it ✅ inline and link the PR.

> Audited libraries: candy-core, candy-sprinkles, sugar-bits, candy-shell,
> candy-shine, sugar-prompt, sugar-charts, honey-bounce, candy-zone.

> Wider port wave (super-candy, sugar-tick, candy-mines, sugar-stash,

> candy-freeze, candy-mold, candy-kit, candy-metrics, sugar-crush,

## CandyCore (← bubbletea)

### API gaps

- ✅ `Program::wait()`, `Program::kill()` — block-until-finished and hard-kill methods distinct from the existing `quit()`. (Backlog #2)
- ✅ `Program::releaseTerminal()` / `restoreTerminal()` for shelling out to `$EDITOR`-style external commands. (Backlog #2)
- ✅ `Program::println(...)` / `Program::printf(...)` as direct methods (we only have the `Cmd::println` / `Cmd::printf` form). (Backlog #2)
- ✅ `ProgramOptions::withoutSignalHandler` shipped — when true, the
  runtime skips installing handlers for SIGINT / SIGWINCH / SIGTSTP /
  SIGCONT regardless of `catchInterrupts`. Use this when the calling
  process already owns signal handling. (Backlog: CandyCore audit gap.)
- ✅ `Msg\CapabilityMsg` shipped — generic capability-reply Msg with `$capability` (caller-supplied identifier) + `$response` (raw payload). Distinct from the specialised `TerminalVersionMsg` (XTVERSION) / `ModeReportMsg` (DECRQM) replies. Use it for capability queries that don't fit those two shapes.
- ✅ **KeyType enum extended** (Backlog #1) — keypad (`Kp0`–`Kp9`, `KpEnter`,
  `KpDecimal`/`Divide`/`Multiply`/`Subtract`/`Add`/`Equal`/`Separator`,
  `KpLeft`/`Right`/`Up`/`Down`/`PageUp`/`PageDown`/`Home`/`End`/`Insert`/`Delete`/`Begin`),
  media (`MediaPlay`/`Pause`/`PlayPause`/`Reverse`/`Stop`/`FastForward`/`Rewind`/`Next`/`Prev`/`Record`),
  volume/mute (`LowerVolume`/`RaiseVolume`/`MuteVolume`),
  locks (`CapsLock`/`ScrollLock`/`NumLock`),
  system (`PrintScreen`/`Pause`/`Menu`),
  modifier-as-key (`LeftShift`/`RightShift`/`LeftCtrl`/`RightCtrl`/`LeftAlt`/`RightAlt`/`LeftSuper`/`RightSuper`/`LeftHyper`/`RightHyper`/`LeftMeta`/`RightMeta`/`IsoLevel3Shift`/`IsoLevel5Shift`),
  `Extended`/`Begin`/`Find`/`Select`, and F-keys F13–F63.
  `KeyType::kittyFunctionalKeys()` exposes the PUA codepoint table; the
  Kitty parser in `InputReader::decodeKittyKey()` consults it.

## CandySprinkles (← lipgloss)

### API gaps

- ✅ `Style::hyperlink()` (OSC-8) shipped. (Backlog #5) Wraps the
  rendered output in `\x1b]8;[id];URL\x1b\\…\x1b]8;;\x1b\\` via
  `Util\Ansi::hyperlink`. Pairs with `getHyperlink()` getter and
  `unsetHyperlink()` reset. Empty-URL form is an alias for unset.
- ✅ `Style::underlineColor()` + `getUnderlineColor()` shipped — emits
  SGR 58 (xterm-256 extension) when underlining is active and a colour
  is set. (Backlog #5)
- ✅ `Style::setString()` + `value()` accessor shipped — bind a
  default content string into the style; `render()` with no arg uses
  it. (Backlog #5)
- ✅ `Style::underlineStyle(UnderlineStyle)` + `getUnderlineStyle()` shipped — `UnderlineStyle::Single` / `Double` / `Curly` / `Dotted` / `Dashed` emit SGR `4:N` sub-style; plain underline keeps SGR 4. (Backlog #5 wrap-up.)
- ✅ `Style::paddingChar(string)` / `getPaddingChar()` and `Style::marginChar(string)` / `getMarginChar()` shipped — fill glyph for padding / margin cells, default `' '`; empty string falls back to space. (Backlog #5 wrap-up.)
- ✅ `Style::borderForegroundBlend(Color $start, Color $end)` shipped — picks four blended stops (t = 0 / 1/3 / 2/3 / 1) and seeds them onto the four border sides via `borderSideFg`. (Backlog #5 wrap-up.)
- ✅ Style unsetters shipped: `unsetHyperlink`, `unsetPadding`,
  `unsetMargin`, `unsetTabWidth`, `unsetAlign`, `unsetVerticalAlign`,
  `unsetInline`, `unsetMarginBackground`, `unsetColorWhitespace`,
  per-side `unsetBorderTop` / `unsetBorderRight` / `unsetBorderBottom`
  / `unsetBorderLeft`. (Backlog #5)
  ✅ `getUnderlineStyle()` shipped (returns `UnderlineStyle::None` when unset).
- ✅ `Border::markdownBorder()` factory shipped (pipes for verticals,
  dashes for horizontals). The inner-piece fields (`middleLeft`,
  `middleRight`, `middle`, `middleTop`, `middleBottom`) already
  existed on `Border`; they're populated for every existing factory
  including the new markdown one.
- ✅ `Color::darken()`, `lighten()`, `complementary()`, `alpha()`, `blend1D()`, `blend2D()`,
  named ANSI constants (`Black`, `Red`, …, `BrightWhite`), top-level
  `hasDarkBackground()` helper. (Backlog #4)
  Color manipulation helpers already lived on `Util\Color`; the named
  16-slot palette + `hasDarkBackground` lipgloss top-level shipped via
  `Sprinkles\Palette`.
- ✅ Custom-renderer abstraction shipped — `Sprinkles\Renderer` value object with `withColorProfile` / `withHasDarkBackground` / `newStyle()` / `lightDark()` / `resolveAdaptive()` / `fromEnvironment()`. (Backlog #6) PHP's stream model is coarser than Go's, so the writer-binding `NewRenderer(out)` shape is not mirrored — pair with `Sprinkles\Output::fprint($stream, ...)` when you need to write to a specific stream.
- ✅ Top-level package functions `Print` / `Println` / `Sprint` / `Fprint` / `Printf` (we only have these as `Style` instance methods). (Backlog #4 follow-up)
  Shipped as `Sprinkles\Output::print/println/sprint/fprint/printf` — style-agnostic helpers mirroring the lipgloss top-level surface.
- ✅ Tree: `offset(start, end)` / `offsetStart` / `offsetEnd` slicing,
  `value()` getter, `Tree::rootOf(label)` named-ctor distinct from
  `Tree::new()->root()`. (Backlog #5 wrap-up.)
- ✅ List: variadic `ItemList::new(items...)` shortcut shipped, plus `Listing\Enumerator::tree()` — list-as-tree enumerator that emits `├─` / `└─` branch glyphs without the full `Tree\Tree` nesting. (Backlog #5 wrap-up.)
- ✅ Table: `clearRows()` + `data(Data)` shipped, plus the `Data` interface (`rows()` / `columns()` / `at($row, $col)`) and a default `StringData` implementation with `fromMatrix(iterable)` / `append()` / `empty()`. Ragged rows pad to the widest column count; non-string cells coerce via `(string)`. (Backlog #5 wrap-up.)
