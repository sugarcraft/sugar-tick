# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[pattern:candy-sprinkles]** Theme's `primary`/`secondary` and `accent`/`muted` slots are passed the **same** Color value in all 10 named constructors — they are true aliases, not distinct colour choices. Consumers reading `$theme->accent` get `$theme->primary`. Do not add new semantics to one without the other.
- **[pattern:candy-sprinkles]** Theme `adaptive()` uses `$bg >= 8 ? light() : dark()` where 8 is the boundary between ANSI bright-black (dark bg) and the first light ANSI colour. This matches common terminal emulator behaviour for COLORFGBG.
- **[pattern:candy-sprinkles]** Theme is the SSOT for theming across consumer libs (sugar-dash, sugar-charts in Phase 03). Any new colour-slot addition must be propagated to all named constructors — there is no default fallthrough; omitting a slot from a constructor leaves it at `\x00` (no colour).
- **[pattern:color-parse-named]** `Color::parse($name)` accepts 38 CSS/ANSI color names (black, red, green, yellow, blue, magenta, cyan, white, bright-* variants, purple/fuchsia/aqua/lime/maroon/olive/navy/teal/silver/gray/grey/orange/pink/brown/coral/gold/indigo/violet/turquoise/salmon/khaki/beige/ivory/lavender/plum/crimson/darkgreen). Case-insensitive, trims whitespace. Throws `InvalidArgumentException` for unknown names. Designed for user-facing color pickers where named colours are more ergonomic than hex codes.
- **[pattern:hsl-factory]** `Hsl::color($h, $s, $l)` uses CSS-style notation (hue 0-360, saturation/lightness 0-100) and delegates to `Color::hsl()` (which accepts 0-1 fractions internally). `Hsl::parse()` handles `hsl(...)` string input including `%` suffixes and whitespace — returns `null` on parse failure rather than throwing, which makes it safe for gradual colour-field enrichment.
- **[pattern:markup-tag-pair]** `Markup::parse($input, $defaultStyle)` parses `[tag]text[/]` syntax — distinct from `StyleParser`'s `[text](fg:red,bold)` format. The markup parser uses a style stack to support nested regions, and returns `list<Cell>` rather than a string, making it suitable for complex multi-style text layout at cell granularity. Unknown color names in tags are silently ignored (no exception).
- **[pattern:layout-spacing]** `Layout::joinHorizontalWithSpacing($pos, $spacing, ...$blocks)` and `Layout::joinVerticalWithSpacing($pos, $spacing, ...$blocks)` accept a `$spacing` parameter (null = no gap, int = gap in cells/rows). The spacing is inserted between blocks but not before the first or after the last. Use cases: dashboards with visible gutters, form layouts with field separators, log viewers with blank-line separation between entries.
- **[pattern:border-gradient-blend]** `BorderGradientBlend::fromColors(Color ...$colors)` accepts 1–5 colors and returns an object whose `sides()` method yields 4 interpolated colors (top / right / bottom / left) distributed proportionally around the perimeter. A single color produces four identical sides; 2–5 colors produce a clockwise gradient. The interpolation uses linear LERP via `Color::blend()`. Use when applying a multi-color border gradient via `Style::borderColor()`.
- **[pattern:style-patch]** `Style::patch($other)` performs an incremental merge: only properties that are explicitly set in `$other` are applied to the receiver — null/[]/false values and unset properties in `$other` are skipped, leaving the receiver's values intact. This differs from `inherit()` which always takes the parent's value when the child hasn't set a property. Mirrors lipgloss v2's `Style.Patch()`.
- **[exception:style-with]** `Style` uses `with()` instead of `mutate()` because it needs the sentinel-bool + `propsSet` tracking pattern: every setter passes `field: $val, fieldSet: true, propsAdded: ['field']` to track which properties the caller explicitly set. This allows `inherit()` to correctly take the parent's value only for properties the child hasn't explicitly set. See `candy-core/src/Concerns/Mutable.php` for the standard trait; this exception is documented because the sentinel-bool pattern cannot be expressed via the simple array-merge mutate approach.

- Lang class now extends `SugarCraft\Core\I18n\Lang` — `t()` method inherited from base; NAMESPACE and DIR are the only per-lib constants.
